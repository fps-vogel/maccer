māccer

A tool for macronizing Latin text.

N.B. This tool is still in development. The word key is under revision, and the full offline version of the program, which can handle longer texts and includes all data and source code, is not available yet for download since I am still writing documentation for it.

Instructions

  1. Paste or write plain Latin text into the box above.
  2. Click māc!.
  3. Insert macrons where missing (✖) or uncertain (✒, ✪, ❡) by placing a hyphen (-) after each vowel that should be marked as long, then click a- › ā.
  4. Remove flags by clicking flags.

To contact me: Click ✉. If you have Latin text with macrons and would like to make this tool smarter, please let me know!

To report a problem: Select your text, click ✉, send, and I will look into it. If your problem is with anything other than macronize, please make a note of it in the email.
FLAGS
✖ unknown word.
✒ ambiguous: uncertain vowels marked with a tilde (~).
✪ guessed based on frequency.
❡ prefix or enclitic detected attached to a known word.
☛ invalid characters detected.

Tips

  • Text can be de-macronized by clicking ō › o.
  • If any text is selected, commands will be applied only to the selection.
  • Click ↺ to undo the most recent command, but this will also undo all changes since then. Click it again to redo the command and all changes after it.
  • Use comments (C-style // or /* */) to exclude any words that should not be macronized, e.g. a passage in another language.
  • Markdown formatting is supported. Click the preview icon in the text area to render.

Q & A

A lot of words get “✖” flags instead of macrons. This program is stupid!

Maccer operates based on a list of Latin word forms generated from digital texts already marked with macrons. It does not generate inflected forms based on principal parts. This has the advantage of not inventing forms that were not actually used, but also the disadvantage of the word list being limited by the number of its source texts, which so far is very small. This may change in the future, but for the sake of simplicity that is the method I chose to start with. That means that while individual forms can be added manually to the word key, e.g. abiēgnō, entire words in all their inflections cannot be added in one stroke, as in abiēgnus, -a, -um.

Consequently, Maccer inserts macrons were possible (i.e. for forms already in the key with no similar possibilities as in hōc and hoc), but it is up to you to take care of the word forms Maccer has guessed or does not know.

The good news is that the more Maccer is used, the smarter it will become. After you have filled in the gaps by hand, Maccer can then analyze your output and add the previously unknown word forms to the key. Ideally the online version would allow anyone to improve the word key in this way, but as I'm not a web programmer, the only way to do this is by sending me texts with macrons (whether created with the help of Maccer or not), and I'll merge them into the word key in the offline version and then update the key in the online version.

So this word key is based on online texts? But those are full of errors!

Yes, the texts I have found that have vowel quantity marked range from almost perfect to worse than nothing, and it would be impossible to correct every single mistake in them. However, with the help of another script that detects possible errors in the word key, and some human intervention, this automated process becomes much less haphazard: for example, hidden long vowels, non-classical orthography, and words incorrectly marked where the correct form is known (e.g. amicus and amīcus) are all detected and corrected under the supervision of the user.

I can’t copy text from Word without losing all the formatting!

Web applications have trouble reading Word formatting, so here it is not supported. But most formatting can be preserved by following these steps:

  1. Plug your Word file into the Word to Markdown Converter.
  2. Copy the resulting Markdown-formatted text, paste it above, and macronize it.
  3. Generate formatting via the preview button, or for more customizability, via Pandoc.

This program is really slow!

I’m not a programmer, least of all a web programmer: I did not imagine Maccer running on a webpage until late in its development, when I found out Lua scripts could be run on the web via the Lua VM. I thought it would be worth a try, and here is the result. The web version would be faster if it were ported to pure JavaScript, but it is what it is.

This program doesn't work on [insert book title here]!

The web version simply cannot macronize texts of about 2000 words or more. The offline version has no such limitation.
